# Contributing Guide

## Branch Strategy
- main → Production
- staging → Staging
- develop → Development
- Feature branches from develop

## How to Contribute
1. Create a feature branch from develop
2. Make your changes
3. Test locally with lutter test
4. Create a Pull Request
5. Wait for review and approval
6. Merge to develop

## Commit Messages
Use clear, descriptive commit messages:
- eat: Add new feature
- ix: Fix bug in X
- docs: Update documentation
- 	est: Add tests for X

## Code Style
Follow Flutter/Dart conventions and best practices.
